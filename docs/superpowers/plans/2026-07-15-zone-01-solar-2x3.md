# Solar Wing 2x3 Asset Sheet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce, split, validate, register, and deliver the first non-character solar-wing asset sheet as a versioned six-asset 2×3 set.

**Architecture:** A private source crop guides one Image Gen chroma sheet. A deterministic local splitter creates six child PNGs, then alpha/cell-bound validation records their proof. Only children, not a recomposed parent, are registered for future runtime placement. The sheet and proof are sent to AYANO after the zone commit is pushed.

**Tech Stack:** built-in Image Gen, Python/Pillow, JSON manifests, Node validation, GitHub, AYANO Discord bot.

- [ ] **Step 2: Run the test to verify the splitter is absent**

- [ ] **Step 3: Implement chroma removal and row-major 2×3 splitting**

The script must calculate proportional 2×3 boundaries, then prefer a detected
fully transparent chroma gutter nearest each internal boundary (the standard
1536×1024 master can therefore use its true blank row/column gaps rather than
cutting through an object). It removes only the sampled border chroma, emits
six RGBA cells, and writes a JSON report with cell bboxes, nontransparent bbox,
alpha edge flags, and pixel count.

- [ ] **Step 4: Run unit and real-sheet validation**

Run: `python3 -m unittest tests/test_split_zone_2x3_sheet.py -v && python3 scripts/split-zone-2x3-sheet.py --input assets/intermediate/zone-01-solar-wing/solar-wing-2x3-v1-chroma.png --out assets/generated/zones/solar-wing-v1 --report proof/intermediate/zone-01-solar-wing-split-v1.json`
