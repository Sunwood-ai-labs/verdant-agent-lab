# Zone 01 — solar wing, 2x3 sheet prompt v1

Use case: stylized-concept

Asset type: reusable HTML pixel-office sprites, generated as one landscape
two-column by three-row chroma-key source sheet.

Input image: the user-supplied office reference, specifically the north-west
solar-wing crop at original-image bbox `[42, 36, 218, 461]`; use it only for
pixel-art style, materials, viewing angle, colours, and object design.

Primary request: create exactly six separate non-character pixel-art objects
arranged in a strict 2 columns × 3 rows landscape sprite sheet, reading
left-to-right then top-to-bottom:

1. empty stone-and-wood mounting frame for the solar array
2. one blue rectangular solar module, front-on and reusable six times
3. blank dark solar-output dashboard panel, no readable numbers or labels
4. one black server/battery rack, front-on and reusable three times
5. long low green control-base device
6. short stone-wall edge with climbing green vine

Style/medium: detailed warm isometric pixel art matching the provided office
reference, sharp readable pixels, clean opaque object silhouettes.

Scene/backdrop: a perfectly flat, single-colour `#ff00ff` chroma-key
background. It must be uniform from edge to edge.

Composition/framing: each object must stay completely inside its own cell.
Leave wide blank gutters between all cells; no cell border lines. Include
generous chroma padding around every object. No object may touch or overlap a
neighbouring cell.

Constraints: no characters or people; no floor tiles, room wall, garden
background, cast shadows, contact shadows, reflection, watermark, logo,
readable text, invented numerals, or UI. Do not combine the six objects into a
scene. Do not include more than six objects.

Avoid: photorealism, anti-aliased painted style, sprite labels, grid labels,
text, duplicate hidden objects, cropped silhouettes, or `#ff00ff` inside any
object.
